# 移动端新版 UI 需求说明

## 核心需求

新增一套移动端 UI，不直接改造现有旧页面。用户可以在旧 UI 和新 UI 之间切换，切换结果写入本地缓存，后续进入应用时自动进入上次使用的 UI 类型。

## 页面范围

### 登录页

- 保留账号、密码、登录、注册能力。
- 保留主题切换、后台入口。
- 注册入口下方不再展示文档站、GitHub、页脚链接或其他说明内容。
- 移动端视觉需要减少右上角工具拥挤感，页面内容到注册入口为止自然收束。

### 首页/账本日历

- 作为新 UI 默认首页。
- 展示当前账本、月份、月收入、月支出、结余、预算进度、日历和今日流水预览。
- 保留语音记账能力，入口应在首页明显但不遮挡日历。

### 快速记账

- 使用移动端底部抽屉或独立记账面板。
- 默认展示高频字段：金额、流水类型、分类、支付/收款方式、日期。
- 低频字段折叠到更多信息：归属、名称、备注。
- 支持保存和保存并继续。

### 流水列表

- 改为移动端日期分组列表，不使用桌面表格压缩。
- 支持搜索、筛选、导入导出、批量操作入口。
- 支出、收入、不计收支颜色和符号必须稳定。

### 月度分析

- 保留月度收入、支出、结余、不计收支。
- 保留消费类型、支付方式、归属、每日流水、每月流水等图表能力。
- 保留 AI 流水分析入口。
- 图表在移动端纵向分块展示。

### 预算管理

- 展示月度预算、固定支出、可用额度、已用额度、剩余额度。
- 固定支出以移动卡片管理，不使用表格。

### 待收款

- 展示待收款项、待收金额、已收款项、已收金额。
- 支持状态筛选、搜索、添加、编辑、收款、删除。
- 逾期状态需要明显。

### 账本管理

- 支持新增账本、添加共享账本、切换账本、编辑、分享、复制分享 Key、删除。
- 当前账本需要有明显标识。

### 类型/默认项/CSV 映射

- 类型管理、默认记账项、CSV 导入映射配置合并为“记账配置”信息架构。
- 类型管理保留同步修改关联流水的风险提示。
- 默认项包括默认流水类型、默认归属、默认支付方式。

### 我的/设置

- 作为低频功能入口。
- 包含账本管理、待收款、预算管理、类型管理、默认记账项、CSV 导入映射、深色模式、修改密码、后台管理、退出登录。
- 需要展示当前 UI 模式，并提供切换到旧 UI 的入口。

## 新旧 UI 切换需求

- 旧 UI 页面上需要有“切换到新版移动 UI”的入口。
- 新 UI 页面上需要有“切换到旧版 UI”的入口。
- 切换后立即写入本地缓存。
- 再次进入应用时，自动进入上次选择的 UI 类型。
- 本地缓存 Key 建议为 `cashbook.ui.mode`。
- 可选值：
  - `legacy`：旧 UI。
  - `mobile-v2`：新版移动 UI。
- 未命中缓存时默认使用 `legacy`，避免对既有用户产生突变。

## 非目标

- 不删除或重构旧页面。
- 不调整后端 API。
- 不改变现有业务数据结构。
- 不在第一版移动 UI 中完整复刻桌面端 CSV 预览表格、批量编辑表格等重型操作；这些能力保留旧 UI 入口。

## 验收标准

- 每个目标页面都有独立 UI 图。
- 有明确的新旧 UI 并存方案。
- 有明确的路由映射、缓存策略、切换入口设计。
- 后续实现时旧页面仍可直接访问。
- 用户切换 UI 后刷新页面、重新进入应用都能保持上次选择。
