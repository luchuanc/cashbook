## 架构复查

### 首轮独立架构复查

结果：不通过，存在阻塞问题。

阻塞问题与处理：

- `/m/*` 未登录 SSR 直达会回落到旧版 `/login`：已修复 `app/middleware/auth.ts`，优先根据目标路由是否为 `/m/*` 决定登录页。
- 新版首页无账本时会请求 `daily()`：已修复 `app/pages/m/calendar.client.vue`，无 `bookId` 时清空首页数据并跳过账本作用域 API。
- 快速记账无账本时可能尝试写入：已修复 `app/components/mobile-v2/QuickFlowSheet.vue`，无 `bookId` 时提示先选择账本并中断保存。

### 回归架构复查

结果：有条件通过，存在非阻塞建议。

结论：

- 首轮 3 个阻塞项已解除。
- 复查指出 `app/pages/m/config.client.vue` 的 `loadPayOptions()` 在无账本状态下仍可能请求账本作用域接口；已补充 `bookId` guard 并在无账本时清空 `payTypeOptions`。
- 其余建议保留为后续优化：统一移动端无账本缺省态、查询参数在新旧 UI 切换中的保留。
- 已进一步处理 `doApi` 会话失效登录一致性：通用 400 拦截会根据当前 `/m/*` 路由或 `cashbook.ui.mode=mobile-v2` 跳转 `/m/login`，其他情况仍跳转 `/login`。
