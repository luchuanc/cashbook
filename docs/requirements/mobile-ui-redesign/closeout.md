# 移动端新版 UI 收尾记录

## 当前状态

实现已完成到可本地构建和关键路由访问的状态，需求流程已完成代码、文档和架构复查闭环。

剩余限制：

- Standard 级需求首轮独立架构子代理复查结论为“不通过：存在阻塞问题”；阻塞项已修复。回归架构复查结论为“有条件通过：存在非阻塞建议”，关键非阻塞建议已处理。
- 当前数据库连接权限受限，无法完成登录后真实数据 CRUD 的端到端验证。
- 语音记账依赖麦克风权限和浏览器/原生录音能力，仍需在真机或可用麦克风浏览器环境复验。

## 已落地范围

- 新增新版移动 UI 路由：`/m/login`、`/m/calendar`、`/m/flows`、`/m/analysis`、`/m/budget`、`/m/receivable`、`/m/books`、`/m/config`、`/m/settings`。
- 新增 `mobile-v2` 布局和移动端组件目录 `app/components/mobile-v2/`。
- 新增 UI 模式缓存 `cashbook.ui.mode`，支持 `legacy` 与 `mobile-v2`。
- 旧 UI 保留原路由，并在 Header 增加切换到新版移动 UI 的入口。
- 新 UI 页面顶部和我的页提供切换到旧版 UI 的入口。
- 新版登录/注册页已按补充要求移除注册按钮下方内容。
- 首页已接入真实月度统计、日历、今日流水、预算进度、快速记账和语音记账。
- 快速记账已使用移动端底部抽屉，并支持语音解析结果预填。
- 已补充无账本边界防护，避免未选择账本时直接请求账本作用域 API 或写入流水。
- 流水、分析、预算、待收款、账本、记账配置、我的设置均已实现移动端页面。
- 复杂 CSV 预览、导入导出和批量表格能力保留旧版入口。

## 验证摘要

- `npm run build`：通过。
- 本地 dev server：启动成功，Nuxt 使用 `http://0.0.0.0:3000/`。
- 路由返回验证：
  - `/m/login` 返回 `200 text/html;charset=utf-8`。
  - `/login` 返回 `200 text/html;charset=utf-8`。
  - `/m/calendar` 返回 `200 text/html;charset=utf-8`。

## 待执行门禁

1. 在可用数据库环境中验证登录后主要 CRUD：快速记账、流水删除、预算保存、待收款处理、账本切换、配置保存。
2. 在移动设备或可用麦克风浏览器中复验语音记账长按、授权、识别、预填和自动写入。
3. 后续优化移动端统一无账本缺省态，以及新旧 UI 切换时保留查询参数。

## 发布建议

- 当前适合进入内部自测，不建议直接发布给全部用户。
- 由于新 UI 采用旁路路由和本地缓存开关，回退方式是切换 `cashbook.ui.mode=legacy` 或直接访问旧路由。
- 若上线试用，建议只让用户主动从旧 UI 入口切换，避免默认影响既有用户。

## 反馈归类

- 稳定知识候选：新旧 UI 模式使用 `cashbook.ui.mode` 控制，默认 `legacy`；新版移动 UI 统一使用 `/m/*` 路由。
- 流程改进候选：Standard 级需求在用户要求“先出规划和 UI 图”后，仍应立即生成完整过程文档，避免实现后补齐。
- 归档信息：本次数据库权限和本地 curl 沙箱限制属于当前环境问题，不作为长期业务规则。
